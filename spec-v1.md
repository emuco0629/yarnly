# アプリケーション仕様書：Yarnly

**バージョン:** v1.0（最終版 — Claude Code実装段階へ移行可能）
**起草日:** 2026年5月17日〜19日
**作成経緯:** Claude + Gemini + sachiyoによる共同設計

---

## 1. 概要と世界観

### コアコンセプト

毛糸玉のキャラクターが歩きながら、頭上に言葉の糸を紡いでいく日記アプリ。たまに、誰かとすれ違う。

### UI/UXのトーン

静寂、ミニマル、無機質ではない温かみ。ホーム画面やタイムラインの概念を持たない。アプリを開くと、そこはもう「外」か「自室」のどちらかである。

### 文字の表現（のろし）

ユーザーが入力フォームで文章を書き送信すると、画面下の毛糸玉から文字が一文字ずつ上へ紡がれていく。文字は緩やかなsin波で左右にうねりながら上昇し、一本の連なった糸のパスを描く。次の投稿は前の糸の続きから始まる。途切れない一本道。

のろしはその人だけのもの。他者ののろしと絡み合ったり、編まれたりはしない。すれ違っても、ただそれぞれのリズムで揺らいでいるだけ。

### テキストの表示ルール

**天井テキスト（全シーン共通）:**
- 画面上部に天井テキストの最新1〜2行が常に表示される
- 上にスクロールして過去の行を読むことはできない
- 自分の言葉が溜まっていくのがリアルタイムで見える
- すれ違い時、他者の色の文字がスッと混入するのもここで視覚的にわかる
- 全体のログは自室に帰ってから初めて読める → 帰宅の動機になる

**散歩モード（外）:**
- 天井1〜2行 + のろしアニメーション + キャラクター + 入力フォーム
- のろしの文字は一定の高さまで表示され、天井テキストに吸い込まれるように消える

**会話モード（リアルタイムチャット中）:**
- キーボード表示中は、天井テキスト部分がそのまま会話ビューになる
- 自分と相手の言葉がリアルタイムで天井に編み込まれていくのが見える
- UIの切り替えは不要。天井が常にそこにある
- LINEと同じ感覚で読み書きできる

**自室モード（中）:**
- 天井テキストの全ログが読める形で表示される
- 多色編みの日記として振り返り・編集できる

### 名前

**Yarnly**（ヤーンリー） — アプリ名であり、キャラクターの名前でもある。毛糸（Yarn）+ ly。「あなたの言葉を紡ぐよ」

### 着想元

- **Bruce Cao氏のTODOアプリ**（@bruce_CQT）— React + p5.js + Claude製。チェック済みタスクが物理演算で崩れ、砂のように積もるUI。のろし表現の起点。
- **日とと記（ひととき）**（diary.aaaaaso.com/hitotoki）— 匿名日記交換サービス。パスキー認証、暴力的コンテンツのフィルタリング。認証・プライバシー設計の参考。

## 2. キャラクター：Yarnly（ヤーンリー）

### デザイン

ユーザーの分身であり、アプリの案内役。SVGで描画。kunkunデザインを参考とした以下の特徴：

- もこもこした凸凹のある丸いシルエット（雲のような輪郭）
- 小さな黒い目（2つ）
- 足（2本、歩行アニメーション付き）
- 手（2本、腕振り・手を振る・会釈・手を挙げるアクション）
- 体表面にジグザグの毛糸テクスチャ

### アクション一覧

- **歩く** — 左へてくてく進む（散歩中）
- **止まる** — 投稿がない時、待機中
- **手を挙げる** — 挨拶（オープニング、他者に話しかける時）
- **会釈（ペコリ）** — すれ違い時のデフォルト動作、会話の切り上げ
- **天井を見上げる** — 自室で日記ログを確認する時
- **椅子に座る** — 自室で日記を書く時

### 色のパーソナライゼーション

初回起動時にオープニング演出で色を選択する（後述）。毛糸玉の体色、のろしの文字色、日記ログの色がすべて連動する。他者の色はそのまま表示される（薄くしない）。

## 3. オープニング演出（初回起動時）

1. **ニュートンのゆりかご** — 上から糸で吊るされた毛糸玉が10個、間隔ぴったりに一列に並んでいる。
2. **色パレット（10色）:**
   - blue `#4A86CF`, orange `#E8722A`, green `#2E8B57`, pink `#D95B8A`, honey `#C4960A`
   - coral red `#D4555E`, lavender `#7B6DAF`, leaf `#5AAD6A`, teal `#1F6B8A`, navy `#2B4E7E`
3. **タッチすると揺れる** — どの玉でも触って揺らせる。迷いながら選べる。
4. **長押しで決定** — 選んだ毛糸玉の糸がプツンと切れて、床に落下・着地する。
5. **キャラクター誕生** — 着地した毛糸玉に足と手と目がつく。手を挙げて挨拶する。
6. **「僕はヤーンリー！あなたの言葉を紡ぐよ」** — 吹き出しまたはテキストで自己紹介。
7. 散歩モードへ遷移。

## 4. 画面遷移とユーザーフロー

アプリは常に散歩モードから始まる。自室モードは「帰る場所」であり、最初の選択肢としては存在しない。

### ① 初回体験（チュートリアル）

説明文ゼロ。体験で全てを理解させる。

オープニング演出の直後、最初の散歩に出発。ユーザーが最初のつぶやきを送信すると、2体のbot Yarnlyと順番に遭遇する。

**1体目：ぶつぶつbot（すれ違い体験）**
- 長めの独り言をのろしに出しながら左から歩いてくる
- 例：「今日の夕飯何にしようかな、昨日カレーだったし、パスタもいいけど冷蔵庫になにがあったっけ」
- 会釈してそのまま通り過ぎる
- → すれ違いと混入を体験する

**2体目：おしゃべりbot（会話体験）**
- 左から歩いてきて、手を振ってくる
- ユーザーが手を振り返すと立ち止まる
- 「はじめまして！散歩気持ちいいね」
- ユーザーが何か打つと反応を返す
- 会釈して去る
- → 手を振る・立ち止まる・会話・切り上げの流れを体験する

**初回の帰宅：**
- 2体目が去った後、扉が現れる
- 自室に入ると、自分の日記の間にぶつぶつbotの断片と、おしゃべりbotとの会話が編み込まれている
- 「編み目を整理しよう」→「このまま」/「編み直す」を体験
- ここで「ほどく」も試せる

一連の体験で、すれ違い・会話・帰宅・振り返り・編集の全機能を一度で理解できる。

### ② bot Yarnlyの常駐

- ユーザーが少ない時間帯にも、bot Yarnlyが散歩している
- 過疎時でも最低限のすれ違い体験を保証する
- botの日記は運営が用意した日常のつぶやき（天気、食事、考え事、季節の話など、長めのぶつぶつ独り言）
- botのYarnlyは固有の色を持ち、通常ユーザーと同じ見た目

### ③ 通常起動時

アプリを開くと白いキャンバスに日付だけが表示される。画面下部にYarnlyが佇んでいる。散歩モードが始まる。

### ④ 入力フォーム

- 画面下部に、行数に応じて自動で縦に広がる拡張型テキストエリアを配置
- フォーム右端にシンプルな送信ボタン（↑）のみ。「編む」ボタンは置かない
- 漢字変換・打ち間違い修正を通常通り行える
- キーボード起動時、毛糸玉はキーボードのすぐ上に押し上げられ、キーボードより上の空間全てがキャンバスとなる

### ⑤ 散歩モード（外）

**開始トリガー:** ユーザーがテキストを送信する（↑ボタン）

**のろしの自動再生:**
- フォームから文字が消え、毛糸玉が画面右から左へ自動で歩行開始
- 頭上の毛糸から、入力した文章が一文字ずつゆっくりと上へ紡がれていくアニメーション再生
- 速度はゆっくり（FLOW_SPEED: 120ms/文字 程度）
- のろしは画面上部の一定の高さまで表示され、そこから先は消えていく（散歩中はスクロール不可）
- テキストは裏側でFirestoreに保存され、自室に戻った時に読める形で表示される
- 続けて次のテキストを入力・送信できる。のろしは前回の続きから伸びる

**すれ違い（非同期）:**
- 直近15〜30分以内に投稿した別ユーザーを1人ピックアップ
- その人の毛糸玉が、のろしを出しながら画面左から右へ歩いてくる
- デフォルト動作：お互い小さく会釈（ペコリ）してそのまま通り過ぎる
- リアルタイムのユーザーも過去のログ（非同期）も、見た目は全く同じ振る舞い。接続状態をユーザーに意識させない
- 1日のすれ違い上限：約5人（溜まるテキスト量に連動）

**テキストの混入:**
- 相手のキャラが画面左端に入場した瞬間から、画面右端に退場するまでの間が混入区間
- その間に相手ののろしが紡いでいたテキストが、自分の日記ログに相手の色で記録される
- 句読点では切らない。画面から出た瞬間で切れる（横糸の途中で切れるイメージ）
- 長い日記の場合、のろしはランダムな途中位置から再生開始（考え事の途中に出くわす感覚）
- 短い日記の場合、最初から再生
- 混入量は、相手の歩行速度 × 画面幅 × のろし再生速度で自然に決まる（1〜2行程度）

**会話モードへの移行（リアルタイム同士のみ）:**
- すれ違いざま「手を振る」（Yarnlyが手を挙げるアニメーション）
- 相手もリアルタイム接続中で「手を振り返す」→ 双方が立ち止まり「会話状態」に移行
- 相手が不在/話したくない → 反応なし → 会釈して通り過ぎる
- 自分が話したくない → 何もしない → お互い会釈して通り過ぎる
- 会話中：ターン制ではなく、双方が自分のペースで入力。会話内容は双方のログに記録される
- 「会釈」（Yarnlyがペコリ）で会話を切り上げ、相手の毛糸玉は歩き去る

**操作は「手を振る」「会釈」の2アクションのみ。**

**散歩の終わり:**
- 毛糸玉はどんどん左に進み続ける（折り返さない）
- 自分のテキスト500文字到達 or 散歩開始から15〜20分経過で、扉が道の先に自動で現れる → Yarnlyが歩いてたどり着き、自動で中に入る
- 途中で帰りたい場合は、常時表示の「🏠」アイコンをタップ → 扉が目の前に現れる → 自動アニメーションで中に入る

### ⑥ 自室モード（中）

**移行演出:**
- 扉が現れ、Yarnlyが自動で中に入る
- 自室には線で描いた窓がある

**帰宅時の演出:**
- Yarnlyが天井を見上げる
- 天井から垂れた糸をYarnlyが引っ張る
- するとテキストログが上から現れる
- 「編み目を整理しよう」

**自室での行動:**
- 散歩で書いた日記の振り返りと編集が主な活動
- 日記の追記（付け足し）もできる
- 誰ともすれ違わない。完全に自分だけの空間
- 他人に自分の言葉が届くこともない

**振り返り・編集:**
- その日の散歩で紡いだ自分の日記テキストがメモ帳のように読める形で蓄積されている
- すれ違った誰かの言葉が、自分のテキストの間に他者の色で挿入されている
- 立ち止まって会話した場合、その会話ログも記されている
- ログ表示後、2つの選択肢が提示される：
  - **「このまま」** → そのまま日記として保存。閉じる
  - **「編み直す」** → 編集モードへ移行
- 編集モード：
  - 自分のテキスト → ✏️マークが現れ、タップで修正可能
  - 他者のテキスト → 〰️「ほどく」マークが現れ、2回タップでほどけて落ちるアニメーションで消える
  - 他者のテキストの書き換えはできない（削除のみ）

**過去の日記：**
- 日ごとに保存
- シンプルなカレンダー（月表示）で遡れる
- 日記がある日に自分の色の●が表示される
- ●をタップするとその日のログが開く
- 負担にならない軽量設計

**散歩に出る:** 🚪（ドア）アイコンをタップ → Yarnlyが扉を開けて外に出る

## 5. 技術要件・アーキテクチャ設計

### ① プラットフォーム & 認証

- **Web (PWA):** Vercelでデプロイ。ホーム画面に追加可能
- **認証:** パスキー（Passkey）認証。パスワード不要、端末保存、匿名性維持

### ② Firebase Realtime Database（RTDB）

**用途:** Presence（存在証明）とリアルタイム同期

**保存データ:** `/presence/{userId}`
```json
{
  "isOnline": true,
  "lastActive": 1716100000000,
  "status": "walking",
  "color": "#b8b0a8"
}
```

**ライフサイクル:**
- 散歩モード開始時にノード作成
- `onDisconnect()` で自動削除（幽霊ユーザー防止）
- 自室モード移行時にも削除

### ③ Cloud Firestore

**用途:** 日記の永続化、すれ違い用プール、タイムラインログ

**データ構造:**

`/yarns/{yarnId}`（すれ違い用の言葉プール）:
```json
{
  "userId": "user_abc",
  "text": "今日はいい天気だった",
  "color": "#b8b0a8",
  "createdAt": "2026-05-19T10:30:00Z",
  "segments": ["今日はいい天気だった"]
}
```
- 直近30分以内のデータのみクエリ対象
- `segments`: 句読点で分割済みの混入用テキスト配列

`/users/{userId}/timeline/{logId}`（自室の振り返り用）:
```json
{
  "type": "self",
  "text": "今日はいい天気だった",
  "color": "#b8b0a8",
  "createdAt": "2026-05-19T10:30:00Z",
  "editable": true
}
```
```json
{
  "type": "mixed",
  "text": "おなかすいた",
  "color": "#c2705a",
  "fromUserId": "user_xyz",
  "createdAt": "2026-05-19T10:32:00Z",
  "editable": false
}
```

### ④ フロントエンド

- **フレームワーク:** React
- **アニメーション:** SVG + `<textPath>` による曲線パスに沿った文字上昇。p5.jsは必要に応じて併用
- **キーボード対策:** `visualViewport` APIでキーボード高さを検出し、毛糸玉をキーボード直上まで押し上げる。キーボードより上の空間全てがキャンバス
- **ホスティング:** Vercel
- **状態管理:** React hooks（useState, useRef）。グローバル状態管理は初期段階では不要

### ⑤ モデレーション

- **初期:** NGワードリストによるフィルタリング
- **中規模以降:** Claude APIで投稿内容チェック（すれ違いで他者に届く分のみ）

## 9. このアプリにしかないもの

日とと記は「静かに交換する」。このアプリは「すれ違いの中で混じる」。

日記が届くのではなく、日記を書いている最中に、誰かの言葉がすこし混じる。ほとんどの日は残像とすれ違うだけだけど、ごくまれに、相手が本当にそこにいる。「あ、いま同じ時間を生きている人がいる」という感覚が、のろしの揺らぎの中で静かに起きる。
